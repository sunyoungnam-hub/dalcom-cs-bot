# 달콤수학 CS 챗봇 기획안

> 1주차 과제 · 달콤교육
> 고객 문의 1차 응대 자동화를 위한 트리구조 챗봇 (HTML 데모)

---

## 1. 역할 (Role)

달콤수학 고객센터의 **1차 상담 도우미** 역할을 한다.
학부모 고객이 자주 묻는 문의를 챗봇이 먼저 안내하고, 챗봇으로 해결되지 않는 경우에만 상담원에게 연결한다. 상담 인력이 단순·반복 문의에 매번 직접 답하지 않도록 돕는 것이 핵심 역할이다.

---

## 2. 목표 (Goal)

- **고객 관점**: 자주 묻는 질문에 대해 빠르고 정확한 답을 즉시 얻는다.
- **운영 관점**: 반복 문의 응대에 드는 CS 리소스를 줄이고, 상담원은 실제로 사람이 개입해야 하는 문의에만 집중한다.
- 4개월간 누적된 실제 문의 데이터를 기반으로, 현실에서 자주 들어오는 문의 유형을 그대로 반영한다.

---

## 3. 입력 (Input)

- **데이터 출처**: 4개월치 CS 문의 누적 데이터 (`달콤교육_cs_로그.xlsx`, 약 2,200건)
- **데이터 구조**: 문의유형(대분류) → 태그1(중분류) → 태그2(소분류) → 질문 요약 / 자동화 가능 여부
- **채널**: 카카오톡 채널, 자사몰 Q&A, 스마트스토어(톡톡·상품 Q&A) 등에서 수집된 문의
- 현재는 엑셀 데이터를 수동 취합하여 사용하며, 추후 각 플랫폼 API 연동 시 자동 갱신 구조로 확장 가능하다.

---

## 4. 출력 (Output)

**결과물**: 단일 HTML 파일로 동작하는 트리구조 챗봇 (`dalcom_chatbot_v1.html`)

고객이 문의에 접근하는 방식은 2가지를 제공한다.

1. **메뉴에서 선택하기** — 대분류 → 중분류 → 소분류를 버튼으로 타고 들어가 답변 도달
2. **직접 질문 입력하기** — 자유 텍스트 입력 시 키워드로 매칭하여 해당 답변으로 안내

**주요 분류 구조**

- 대분류: 수강 / 일정 / 환불 / 구입 / 배송 / 기타
- 강의·일정 문의 시 강의 종류를 별도로 분류
  - **프로젝트 (5개)**: 유아사고력 1·2단계, 초등사고력 1·2·3단계 프로젝트
  - **특강 (7개)**: 수배열판 / 직산 / 소마큐브 / 분수소수 / 곱셈나눗셈 / 유아 로드맵 / 초등 로드맵 특강
- 강의 일정·특강 정보는 "자사몰(홈페이지)에서 확인 안내"로 응답

---

## 5. 제약 (Constraints)

- **민감정보는 절대 답변하지 않는다.** 다음 항목이 질문에 포함되면 답변 대신 안내 문구를 띄우고 상담원 연결로만 유도한다.
  - 사업자번호, 대표자 개인정보
  - 수강생 명단 및 개인정보(이름·연락처·주소), 수강 인원 수
  - 내부 운영 비용·수수료, 강사 개인 연락처, 미공개 커리큘럼·단가
  - 타 경쟁 교육업체 비교, 환불 금액 구체적 계산
  - 법적 분쟁 관련 문의, 미공개 신규 과정 정보
- 금지 항목 안내 문구: **"이 내용은 챗봇에서 답변드릴 수 없는 항목이에요. 상담이 필요하시면 아래 상담원 연결을 이용해 주세요."**
- 상담원 연결은 자동으로 넘기지 않고, **고객이 직접 버튼을 눌러야** 연결된다.
- 말투는 학부모 대상으로 친근하되 과하게 AI스럽지 않게, 이모티콘은 최소화한다.

---

## 6. 검수 기준 (Verification)

- 답변이 해결되지 않을 경우(고객이 "아직 안 됐어요" 선택), **3회까지는 다시 안내**하고 **4회째에 상담원 연결**을 제공한다. (불필요한 상담원 연결을 줄이기 위함)
- 트리구조는 최소 단계를 거친 뒤에 상담원 연결이 노출되도록 한다. (첫 답변에서 바로 상담원으로 빠지지 않음)
- 금지 키워드 입력 시 답변이 차단되고 안내 문구만 노출되는지 확인한다.
- 강의 분류 선택 시 정확한 강의명이 전달되고, 엉뚱한 답변이 나오지 않는지 확인한다.

---

## 참고 사항

- 본 결과물은 **HTML 데모 버전**으로, 실제 카카오톡 채널 등에 연동하려면 별도의 API 연동·배포 단계가 필요하다.
- 답변 문구는 누적 데이터 패턴을 기반으로 작성되었으며, 실제 운영 적용 전 달콤수학 정책(환불 규정, 재수강 할인 등)에 맞춘 검수가 필요하다.
- 데이터가 업데이트되면 재생성 스크립트(`build_tree.py`)를 통해 트리를 다시 생성할 수 있다.
